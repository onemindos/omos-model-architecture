The omd Binary — One Artifact, Every Device

One Go binary cross-compiled for every target. Deploy it, configure the mode, it connects to the hub. No runtime. No deps. 5MB static.

# Cross-compile for every target GOOS=linux GOARCH=arm64 go build -o omd-linux-arm64 → Pi, drone, robot GOOS=linux GOARCH=amd64 go build -o omd-linux-amd64 → VPS, workstation GOOS=darwin GOARCH=arm64 go build -o omd-mac-arm64 → your Mac GOOS=windows GOARCH=amd64 go build -o omd-windows.exe → Windows nodes # Deploy and run omd --mode=edge # embedded NATS + local sensor bridge omd --mode=bridge # protocol translator (MAVLink/CAN/BLE → NATS) omd --mode=leaf # leaf node connecting to hub, offline resilient omd --mode=client # lightweight subscriber/publisher only # The binary discovers the hub via NATS leafnode remote # The hub is just a nats-server config — no coordinator needed # $SRV.PING discovers every running service automatically
